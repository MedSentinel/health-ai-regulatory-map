# Example Cases

Version: `v0.1`

These short cases show how to use the repository on real-world style claims.

They are not legal conclusions.
They are editorial analysis exercises.

## Included cases

- [ambient-scribe-patient-centered.md](./ambient-scribe-patient-centered.md): when workflow efficiency is framed as patient benefit
- [midjourney-medical-imaging.md](./midjourney-medical-imaging.md): when a bold imaging narrative outruns visible validation detail
- [uk-ai-sandbox-announcement.md](./uk-ai-sandbox-announcement.md): when innovation language and evidence maturity need to be separated

## How to read these

Each case follows the same structure:

1. the public claim
2. why it matters
3. which lenses to start with
4. which claim-language issues appear
5. what remains unknown

This is the main habit the repository is trying to build:

move from impression to structured questions.
